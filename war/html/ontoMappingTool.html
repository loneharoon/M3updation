Ontology/Dataset Matching Tools

  • Ontology matching web site
  • Ontology Matching Book, Second Edition [Euzenat et al. 2013]
  • Ontology matching: state of the art and future challenges [Shvaiko et al. 2013]
  • Looking for a matching tool available online, easy to test (e.g., Aroma, LogMap), could be integrated as a jar in our framework
  • Test 1: Equivalence (e.g., Snowy, SnowyWeatherState)
  • Test 2: Etymology=stemming (e.g., Rain/Rainy, fog/foggy)
  • Test 3: Synonyms (e.g., Rain/Precipitation)
  • Test 4: Modelling (e.g., driver'state as a concept or as a property)
  • Semantic similarity, string-based similarity, termonological (Jaccard, Leveinstein, SMOA labels and descriptions) fog/foggy
  • Linguistic similarity (dictionnary, Wordnet, synset)
  • Structural similarity (semantics of terms/ontological context):
    • Hyponym (... is a kind of person)
    • Hypernym (person is a kind of ...)
    • Holonym (person is a part of ...)
    • Moronym (part of person)
    • Synonym (e.g., rain/precipitation)
    • Antonym
  • Java API for WordNet searching (JAWS)
  • Wordnet synset for synonyms
  • Differences between usual ontologies and IoT ontologies:
    • No labels or comments
    • No domain or range
    • No attributes for a concept (e.g., Person, name, tel)
    • Need a dictionnary specific to our domain (Wordnet, synset not enough)
    • We do not have the dataset with the ontology
    • Different language
    • Abbreviation (e.g., SPO2)
